## 2025-02-14 - Active states for state-based routing navigation
**Learning:** This application uses pure state-based routing via `<button>` elements instead of standard anchor tags (`<a>`) for its main navigation menu within `Header.tsx`. Because standard link navigation is circumvented, the active routing state must be explicitly communicated to screen readers using `aria-current="page"` tied to the `currentPage` prop. Furthermore, using custom `<button>` routing means standard browser focus styles might be confusing or inadequate if clicking also focuses the element; using Tailwind's `focus-visible` utilities (like `focus-visible:ring-2`) is essential to restrict focus rings strictly to keyboard navigation, preserving a clean UI on mouse clicks.
**Action:** When implementing or modifying custom navigation buttons that use state-based routing, always manually inject `aria-current={isActive ? 'page' : undefined}` and rely on `focus-visible` for keyboard accessibility styles to prevent mouse click visual clutter.
